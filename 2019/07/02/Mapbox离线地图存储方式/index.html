Mapbox离线地图存储方式

标签（空格分隔）： Mapbox

Mapbox的离线地图使用SQLite存储。主要包含六个表：

表名 存储内容
regions 存储离线地图的样式文件，范围，级别等数据
resources 存储样式文件中的字体图片等资源
tiles 存储具体的瓦片数据
region_resources 存储region和resources的关联关系
region_tiles 存储region和tiles的关联关系
sqlite_sequence 存储resources和tiles的统计数据

regions

主要包含以下三个字段：

  • id 区域编码

  • definition 区域定义，是一个JSON字符串

  • description 区域描述,是一个blob类型的字段，存储区域的名称等内容

tiles

tiles存储的是离线瓦片缓存
以下是tiles内部的一些主要字段:
| 字段 | 描述|
| - | :– |
|id | 编码
|url_template| 瓦片下载的模板，这个很重要，是用来确定该瓦片是属于哪个数据源的，如果模板相同，瓦片数据就可以复用。
| z | 瓦片级别
| x |瓦片列号
| y | 瓦片行号
|expires|瓦片过期时间,该数据库主要职业其实是Mapbox的瓦片请求缓存库，因此包含了http请求的本地缓存需要的一些字段,后面要介绍的modified,etag都属于此类;
|etag | 本地缓存标识，下次去网络请求数据时，会携带该数据,服务端生成的etag如何和该值匹配,则返回状态码304,使用本地缓存即可.
|data|具体的瓦片数据
|compressed| 是否压缩
|accessed| 最后访问时间
|must_revalidate|

resources

Mapbox在渲染矢量文件时需要用到一些字体和图片资源，下载离线地图时，这些数据也要下载，resources存储的就是这些数据。

字段 描述
id 编码
url 资源的url
kind 资源类型
expires 过期时间
modified 修改时间
etag Entity tag 实体标识
data 资源数据
compressed 是否压缩
accessed 访问时间
must_revalidate ??

region_resources

只有两个字段:region_id和resource_id，将region和resource关联，删除离线地图时，可以通过这个表格找到对应的资源数据.

region_tiles

只有两个字段:region_id和tile_id，将region和tile关联，删除离线地图时，可以通过这个表格找到对应的瓦片数据.

sqlite_sequence

主要存档离线数据的统计，包含瓦片数目，资源数目和区域数量

Mapbox SDK已经提供了离线地图下载功能，能满足我们的要求；但是遇到特殊情况：如没有在线的地图服务，瓦片数据量太大(Mapbox SDK只支持6000个离线瓦片)的情况下，我们可以绕过MapboxSDK，直接操作数据库，管理离线地图。
